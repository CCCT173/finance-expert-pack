# 融合版股票分析框架 v1.0

> 整合 stock-master（基本面+陷阱检测+三情景）+ 23策略（技术指标+筹码+评分）+ stock_enhanced.py（数据获取）

---

## 一、分析流程

### 阶段1：快速筛选（23策略技术面）

1. **超卖扫描**：J值<20 的标的
2. **MACD信号**：金叉/即将金叉
3. **主力资金**：近3日净流入翻正
4. **布林位置**：触及或接近下轨

通过筛选 → 进入阶段2

### 阶段2：深度验证（stock-master框架）

**Step 1：陷阱检测** ⚠️ 最优先

| 陷阱类型 | 检测方法 | 数据源 |
|----------|----------|--------|
| 财务造假嫌疑 | 经营现金流 vs 净利润长期背离、应收暴增、存货异常 | 财报 |
| 大股东掏空 | 质押率>50%、关联交易高、频繁增减持 | akshare质押/增减持 |
| 估值陷阱 | PE低但ROE持续下滑、周期股在盈利峰值 | 5年ROE趋势 |
| 流动性陷阱 | 日成交额<5000万、股东户数骤降 | 实时行情 |
| 概念炒作 | 营收无变化但股价暴涨、概念板块蹭热点 | 主营构成 |
| 商誉雷 | 商誉/净资产>30%、收购标的业绩变脸 | 财报附注 |

命中任一项 → 标红警告，降低仓位或放弃

**Step 2：基本面速览**

- 最新价/涨跌幅/市值/PE(TTM)/PB/52周高低
- 股息率/ROE/营收增速
- 行业排名（PE/PB/ROE分位）

**Step 3：同行对比**

同行业Top5对比：PE/PB/ROE/营收增速/净利增速
→ 该股定位：龙头/中游/尾部

**Step 4：资金面全景**

| 维度 | 数据 | 判断 |
|------|------|------|
| 主力资金5日 | 超大单/大单/中单/小单拆分 | 机构还是散户在买 |
| 北向资金20日 | 持股变化趋势 | 外资态度 |
| 融资融券 | 余额变化 | 杠杆资金情绪 |
| 股东户数 | 近2季变化 | 筹码集中/分散 |

**Step 5：事件催化日历**

- 近期新闻/公告/研报
- 业绩预告/快报时间
- 行业政策/催化事件
- 限售解禁日期

### 阶段3：综合评判

**评分卡（1-10）**

| 维度 | 评分 | 权重 | 加权 |
|------|------|------|------|
| 基本面（ROE/营收/估值） | | 25% | |
| 技术面（J值/MACD/布林/筹码） | | 25% | |
| 资金面（主力/北向/融资） | | 25% | |
| 陷阱检测（反向扣分） | | 15% | |
| 事件催化（正/负面） | | 10% | |

**三情景分析**

| 情景 | 目标价 | 条件 | 失效位 |
|------|--------|------|--------|
| Bull | | 最有利条件组合 | |
| Base | | 当前趋势延续 | |
| Bear | | 最不利条件组合 | |

**行动标签**（禁止绝对买卖指令）：
- `积累研究` — 需要更多信息
- `持有观察` — 已持仓等信号
- `等回调` — 方向看多但价位不好
- `轻仓试探` — 信号偏多但有不确定性
- `回避至证据改善` — 陷阱检测命中或信号矛盾

## 二、持仓管理模板

### 建仓前必填

```
股票: 
代码: 
建仓价: 
股数: 
总金额: 
止损价: (ATR止损 或 固定百分比)
止盈一: 
止盈二: 
陷阱检测: 通过/命中___
三情景Bull/Base/Bear: ___/___/___
失效条件: 
```

### 持仓跟踪

每次盘中检查必查：

1. 现价 vs 止损/止盈
2. 主力资金今日流向
3. 是否有新事件/公告
4. J值/MACD是否有变化

### 止损规则

- **硬止损**：跌破止损价无条件卖出
- **软止损**：重大利空事件（如业绩变脸、监管处罚），即使未触硬止损也考虑减仓
- **时间止损**：5个交易日未达预期，重新评估是否继续持有

## 三、数据获取清单

### 必查项（每次分析都必须有）

| 数据 | 接口 | TTL |
|------|------|-----|
| 实时行情 | stock_enhanced.py --quick | 60s |
| K线+技术指标 | stock_enhanced.py --tech | 5min |
| 主力资金流向 | stock_enhanced.py --fund | 5min |

### 按需查（深度分析时）

| 数据 | 接口 | TTL |
|------|------|-----|
| 北向资金 | stock_enhanced.py --north | 2h |
| 融资融券 | stock_enhanced.py --margin | 2h |
| 龙虎榜 | stock_enhanced.py --lhb | 2h |
| 机构持仓 | stock_enhanced.py --holder | 24h |
| 同行对比 | web_search + akshare | 24h |
| 财报5年 | akshare stock_financial_analysis_indicator | 24h |
| 新闻/公告 | web_search + akshare stock_news_em | 1h |

### 兜底规则

- 数据缺失 → 标记"未取到"，不猜测
- web_search兜底 → 标记fallback
- 矛盾数据 → 同时呈现，说明来源

## 四、输出纪律

1. **先给结论**：一句话定调 + 行动标签
2. **陷阱检测结果前置**：命中陷阱立即警告
3. **包含"不买/暂缓"理由**
4. **三情景代替单点预测**
5. **结尾给监控点**：具体数据/事件/量价触发条件
6. **矛盾必须呈现**，不和稀泥

---

## 五、与23策略的协作

- **选股信号**：23策略的J值超卖/MACD金叉 → 进入本框架的阶段2
- **技术评分**：23策略的综合评分 → 作为本框架"技术面"维度的输入
- **筹码分析**：23策略的M003/M022 → 补充本框架的资金面维度
- **形态识别**：23策略的M006/M007/M008 → 补充建仓时机判断
